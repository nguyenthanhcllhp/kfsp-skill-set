Chào mừng bạn đến với KFSP Skill Set

Tài liệu onboarding cho Product Owner / Product Manager / Project Manager mới. Đọc xong bạn sẽ hiểu cách team vận hành dự án từ ý tưởng đến release.

🤔 Skill Set là gì?
Bộ 20 công cụ (skills) giúp team (người + AI) chạy dự án có kiểm soát. Mỗi skill giải quyết 1 vấn đề cụ thể: từ dự báo rủi ro trước khi làm, đến kiểm tra chất lượng sau khi build, đến đóng phase sạch sẽ.
👤 Ai dùng?
PM/PO ra lệnh, AI (Claude) thực thi skills tự động. PM chỉ cần: approve plan, test trên device (5-10 phút/build), bấm Đạt/Lỗi. AI lo phần còn lại (85% effort).
🌐 Dùng cho ngành nào?
Thiết kế domain-agnostic. Đang dùng cho app development (Flutter) nhưng framework áp dụng được cho marketing campaigns, event management, content production — bất kỳ dự án có spec → execute → verify.
📐 Triết lý cốt lõi
Mọi thứ là dự án. Roadmap = dự án. Phase = dự án con. Task = dự án nhỏ hơn. Ở MỌI cấp đều cần: Goals → Plan → Execute → Verify → Close. Skills bảo vệ ở mọi cấp.

5 tầng bảo vệ — đọc từ dưới lên

Nếu tầng dưới fail, tầng trên vô nghĩa. T1 (Safety) là nền tảng.

3 tầng test brief

📋 Build Test
5-10 items — chỉ phần thay đổi trong build này. PM test 5-10 phút. Mỗi task 1 build test.
📋 Phase Test
20-30 items — tổng hợp cả phase. Items đã pass → skip. Focus items mới + regression. 30-60 phút.
📋 Release Test
100+ items — full regression 22 features. Performance, accessibility, store compliance. 2-4 giờ (có progress tracking).

Bug flow

PM bấm "Lỗi" → bug-log: BUG-YYYYMMDD-NN → AI fix + regression test → Re-build (chỉ items fixed) → PM re-test 1-2 items → Pass → Close bug
