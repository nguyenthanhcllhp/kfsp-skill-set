Rule 20 — Dual-Session Workflow

💻 Desktop — Code Flutter

  • lib/ Viết code Dart
  • test/ Unit tests
  • build Build APK/IPA
  • Skills: start, check, gate --commit, test --verify

🖥️ Terminal — Rà soát tài liệu

  • docs/ Cập nhật tài liệu
  • test_cases/ Test case registry
  • plans/ Plan files
  • Skills: docs, review, test --brief, risk, log
⚠️ CẤM 2 phiên cùng sửa 1 file. Desktop đụng lib/, Terminal đụng docs/.

8 Skills mới — Chi tiết

1

Bắt đầu phiên

/kfsp:start
Khởi tạo phiên làm việc: kiểm tra git, phát hiện drift code-docs, nhắc bật dual-session.
Gộp từ: session-start
Tự động đầu phiên
2

Rà soát sức khoẻ

/kfsp:check
Đánh giá toàn diện: health score, blast radius sau thay đổi, source sync, token audit.
Gộp từ: guard + sweep + sync-check
--quick --full --tokens --sync
3

Kiểm thử

/kfsp:test
Pipeline test: verify build → tạo brief cho PM → log bug từ PM feedback.
Gộp từ: build-verify + test-brief + bug-log
--verify --brief --bug --bugs
4

Review UX/UI

/kfsp:review
So sánh UI vs Figma (parity) + đánh giá UX Nielsen heuristics.
Gộp từ: ux-parity + ux-audit
--parity --audit
5

Tài liệu

/kfsp:docs
Quản lý tài liệu: status, gap analysis, cập nhật, đóng phase, HDSD readiness.
Gộp từ: doc-pilot + post-phase
--status --update --close-phase --hdsd
6

Cổng chặn

/kfsp:gate
Chặn trước commit (23 checks) hoặc trước release (12 gates). PHẢI pass mới tiếp.
Gộp từ: pre-commit + release-gate
--commit --release
7

Ghi chép

/kfsp:log
Ghi quyết định, sự cố, bài học. Phân tích 5-Whys cho incident. Tìm kiếm precedent.
Gộp từ: dev-journal + incident-review
--log --incident --search --digest
8

Phân tích rủi ro

/kfsp:risk
Dự đoán rủi ro trước khi code + kiểm tra contracts 4 bề mặt + phát hiện regression.
Gộp từ: pre-mortem + surface-test + sentinel
--before --contracts --regression
Bỏ hẳn 2 skill:
product-health — thông tin đã có trong check --full
help — liệt kê 8 lệnh ngay trong CLAUDE.md, không cần skill riêng

🚫 Quy tắc CẤM

  • ❌ Desktop sửa docs/, test_cases/, plans/
  • ❌ Terminal sửa lib/, test/, build files
  • ❌ 2 phiên cùng sửa 1 file (ghi đè nhau)
  • ❌ Terminal commit code (chỉ Desktop commit)

✅ Quy tắc ĐƯỢC

  • ✅ 2 phiên cùng ĐỌC bất kỳ file nào
  • ✅ Terminal đọc lib/ để review/audit
  • ✅ Desktop ghi note để Terminal cập nhật docs
  • ✅ Chạy skill ở phiên nào cũng được (nếu đúng phân vùng)

Vòng đời 1 Phase

1. Mở phiên
/kfsp:start → nhắc bật Terminal → /gsd:discuss-phase
2. Lên kế hoạch
/kfsp:risk --before → /gsd:plan-phase → lưu docs/plans/P{N}.md
3. Code (lặp lại)
Code → /kfsp:check --quick → /kfsp:gate --commit → commit
Terminal song song: /kfsp:docs --update
4. Build & Test
/kfsp:test --verify → /kfsp:test --brief → PM test → /kfsp:test --bug
5. Fix bugs (nếu có)
/kfsp:log --incident (nếu bug nghiêm trọng) → fix → quay lại bước 3
6. Đóng phase
/kfsp:check --full → /kfsp:docs --close-phase → /kfsp:log --log → /gsd:verify-work
7. Release (khi sẵn sàng)
/kfsp:risk --contracts → /kfsp:risk --regression → /kfsp:review --audit → /kfsp:gate --release
