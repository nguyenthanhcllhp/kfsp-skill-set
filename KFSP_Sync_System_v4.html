Rules — Ở đâu, enforce thế nào?

Rule 21 — Change Manifest (Root CLAUDE.md)
Nội dung:
Sau MỖI câu trả lời có code change, PHẢI:
  1. Ghi entry vào .changes.json với impact map
  2. Cuối câu trả lời có section 🔄 Sync
  3. Nhắc Thanh sang Terminal chạy check cụ thể
Vi phạm = DỪNG → ghi manifest → mới được tiếp.
Rule 8 mở rộng — Response Format
Hiện tại (Rule 8):
Mỗi response có: 🥋 Skills đã chạy + 📝 Docs đã cập nhật + 🎨 Convention check

Mở rộng thêm:
🔄 Sync: Thay đổi gì, ảnh hưởng doc nào, Terminal cần chạy gì

Vị trí Rules

RuleỞ đâuAi tuân thủEnforce thế nào

Hỏi — Đáp

📄 .changes.json là 1 file / phiên hay 1 file cho tất cả?
1 file / 1 phiên. Khi /kfsp:start chạy → tạo mới (hoặc reset). Cuối phiên → archive sang docs/logs/changes/changes_YYYY-MM-DD.json. Tránh file phình to, dễ track per-session.
📊 Session HTML là file log chung của cả 2?
Đúng. Desktop ghi entries (code/commit), Terminal ghi results (check/docs update). Cả 2 đọc được. Cuối phiên = báo cáo đầy đủ cho PM review.
⚖️ Rule ghi ở đâu để enforce?
Root /CLAUDE.md — Rule 21. Đây là file BẮT BUỘC mà mọi phiên Claude Code đều đọc trước khi bắt đầu. Không phải memory (có thể quên), không phải plan (có thể bỏ qua). CLAUDE.md = luật.

Thêm vào Rule 8 (Response Format) yêu cầu section 🔄 Sync trong mỗi câu trả lời có code change.
Nếu Thanh quên chuyển sang Terminal thì sao?
Entries tồn đọng trong .changes.json (processed: false). Khi Terminal chạy bất kỳ lúc nào → xử lý tất cả entries chưa processed. Cuối phiên nếu vẫn còn unprocessed → /kfsp:check --full sẽ cảnh báo.
Desktop code liên tục 5 lần không chuyển Terminal?
Vẫn OK. .changes.json tích lũy 5 entries. Khi Terminal chạy 1 lần → xử lý cả 5. Không mất data. Chỉ là docs chậm cập nhật hơn so với code — nhưng cuối phiên vẫn đồng bộ 100%.
Có đảm bảo spec/testcase/doc luôn đồng bộ?
Có, qua 3 lớp bảo vệ:
  1. Lớp 1 — Rule 21: Desktop BẮT BUỘC ghi impact map → Terminal biết CHÍNH XÁC cần cập nhật gì
  2. Lớp 2 — processed flag: Entry chưa xử lý = gap rõ ràng → không bỏ sót
  3. Lớp 3 — /kfsp:check --full cuối phiên: Verify 0 unprocessed entries + docs khớp code
